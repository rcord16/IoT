Announce Count

The number of times between hops that the AP will announce the next hop information (such as frequency) to clients. For example, if the Hop Interval is set to 10000 ms, and the Announce Count is set to 10, then every 1000 ms the AP will send an announcement with upcoming hop information to the clients. The larger the time period between the Announce Count and Hop Interval, the higher the risk of timing drift (hops not being synchronized), so we recommend that you keep the Hop Interval set to 100 ms (or the Announce Count set to 1/100th of the Hop Interval). The default value is 30.
